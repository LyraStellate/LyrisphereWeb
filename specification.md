# **VRChat連携 DJイベントシステム仕様書**

## **1\. システム概要**

本システムは、VRChat内の特定のワールド（イベント会場）と外部Webサーバーを連携させ、参加ユーザー全員の持ち寄ったプレイリストからランダムに楽曲を再生する「参加型DJイベント」を実現するための仕組みである。

ユーザーに煩雑なログイン操作をさせず、シームレスな体験（UX）を提供することを目的とする。

## **2\. 運用前提（重要）**

* **単一インスタンス運用:** 本システムは「イベントオーナーが存在する1つのインスタンス」でのみ稼働させることを前提とする。VRChatの仕様上、外部サーバーからインスタンスを区別できない問題（マルチインスタンス問題）は、この運用ルールによって回避する。
* **カジュアル用途:** 扱うデータは楽曲のプレイリスト等であり、高度な個人情報や決済情報は含まないものとする。

## **3\. システムコンポーネント**

1. **VRChatクライアント (Udon):** ユーザーの認証URL生成、ビート（生存確認）送信、動画プレイヤー（Yama Player）の制御、メタデータのUI表示を行う。
2. **Webサーバー:** ユーザーのマイページ提供、Udonからのビート受信（ログイン状態の管理）、プレイリストからの楽曲抽選・リダイレクト処理を行う。
3. **ユーザーブラウザ (PC/スマホ):** マイページへアクセスし、プレイリストの管理を行う。

## **4\. ユーザー体験（UX）フロー**

### **4.1. 初回参加時のセットアップ**

1. ユーザーがVRChatのイベントワールドにJoinする。
2. ワールド内のUIに、ユーザー名から生成された専用のURL文字列（例: .../api/login?id=\[暗号化ID\]）が表示される。なお、ユーザー名には記号や日本語が含まれる場合があるため、生成はbase64などを用いて暗号化およびurl用に最適化する。
3. ユーザーはワールド内でその文字列をコピーする。
4. コピーした文字列を、ワールド内に用意された VRCUrl InputField にペーストする。
5. Udonは入力された VRCUrl をVRChatの **Persistence (Player Data)** に保存する。（次回以降の入力は不要となる）

### **4.2. マイページでの楽曲登録**

1. ユーザーはコピーしたURLをPCやスマホのブラウザで直接開く。
2. URLに含まれるIDをデコードし、ユーザーが自動判別され、「〇〇さん、おかえりなさい」と短いウェルカムアニメーションが流れ、ログイン不要でマイページが表示される。
3. YouTubeやニコニコ動画のURLをフォルダ分けしてプレイリストに登録する。


### **4.3. イベント中（ワールド内での自動進行）**

1. ログイン状態（ビートを送信中）のユーザーのプレイリストから、サーバーがランダムに1曲を選定する。
2. ワールド内の動画プレイヤーでその楽曲が再生される。
3. 再生と同時に、提供者名や楽曲情報がワールド内のパネルに表示される。
4. 再生終了後、自動的に次の楽曲が選ばれループする。

## **5\. API仕様 (Base URL: https://lyrisphere.lyrastellate.dev)**

### **5.1. ログイン 兼 ビート送信 API**

エンドポイントの数を減らすため、1つのURLで「ブラウザ向け」と「Udon向け」の2つの役割を担う。リクエストの User-Agent で処理を分岐させる。

* **エンドポイント:** GET /api/login?id={暗号化ID}
* **パラメータ:** id (Udonでユーザー名から生成された一意の文字列)
* **処理分岐:**
  * **パターンA (ブラウザからのアクセス):**
    * 条件: User-Agent が一般的なWebブラウザ。
    * 処理: ユーザーのマイページ（HTML）を返す。
  * **パターンB (Udonからのアクセス / ビート):**
    * 条件: User-Agent が VRChat / UnityPlayer 等。
    * 処理: 該当ユーザーを「ログイン中（インスタンス在室）」状態に更新する。
    * レスポンス: データ転送量節約のため、空の 200 OK のみを返す。

### **5.2. 楽曲抽選 API (オーナーからのトリガー)**

* **エンドポイント:** GET /api/set
* **用途:** イベント進行のために、動画終了時などにワールド内のイベントオーナー（Udon）が叩く。
* **処理:**
  1. 現在ログイン状態（直近でビートが届いている）ユーザーのリストを取得。
  2. 該当ユーザー全員のプレイリストからランダムに動画を1つ選択。
  3. 選択した動画情報をデータベースの EventState に記録する。
  4. 処理の成功を返す。

### **5.3. 楽曲リダイレクト API**

* **エンドポイント:** GET /api/get
* **用途:** ワールド内の動画プレイヤー（Yama Player）の入力URLとして指定する。オーナーが動画プレイヤーにこのURLをセットする。
* **処理:**
  1. EventState に記録されている現在の動画情報を取得。
  2. 該当動画の実際のURL（YouTube等）へ HTTP 302 Redirect を返す。

### **5.4. 楽曲メタデータ取得 API**

* **エンドポイント:** GET /api/info
* **用途:** 動画再生開始時にUdon (VRCStringDownloader) から叩く。
* **処理:**
  * 直近で api/get によって選定された動画の情報をJSON形式で返す。
* **レスポンス例:**
  {
    "title": "楽曲のタイトル",
    "composer": "作曲者名",
    "release\_date": "2023-01-01",
    "provider\_name": "プレイリスト提供ユーザー名",
    "folder\_name": "フォルダ名",
    "added_by_users": ["UserA", "UserB"],
    "liked_by_users": ["UserC", "UserD"]
  }

### **5.5. お気に入り登録（すき！） API**

* **エンドポイント:** GET /api/like?id={暗号化ID}
* **用途:** イベント中に再生されている楽曲を自身の「すき！」フォルダに保存する。VRChat内からボタンを押した際にUdonがこのAPIを叩く。
* **処理:**
  1. 現在 `EventState` に記録されている再生中の動画を、アクセスしたユーザーの「すき！」フォルダにコピーする。
  2. 既に登録されている状態でもう一度叩かれた場合は、「すき！」フォルダからその楽曲を削除する（トグル動作）。
  3. VRChatからのリクエストのUser-Agent処理やID復号化はログイン処理と同様。

## **6\. ワールド（Udon）側の実装要件**

1. **ID生成ロジック:** ユーザー名を取得し、特定のルール（または暗号化/ハッシュ化）でID文字列を生成する。
2. **Persistence管理:** VRCUrl InputField の入力をPersistenceに保存・読み込みする。入力ミスに備え、保存データをクリアする「リセットボタン」を実装する。
3. **ビート送信:** Persistenceから読み込んだ VRCUrl を使用し、定期的（例: 60秒ごと）に VRCStringDownloader 等で非同期リクエストを送信する。
4. **動画プレイヤー連携:**
   * 動画の終了イベント（OnVideoEnd）や次へ進む操作を検知し、オーナーが api/set を叩く。
   * その後、オーナーが動画プレイヤーに api/get を読み込ませる（全ユーザーに共有される）。
   * 動画の再生成功イベント（OnVideoReady / OnVideoStart 等）を検知して api/info を叩く。
   * 取得したJSONをパースし、UIテキストに反映する。

## **7\. マイページ（Web）側の実装要件**

フロントエンドは白を基調としたシンプルなデザインにし、AI感を無くしおしゃれで洗練されたデザインにすること。アニメーションを適度に用いてユーザーの操作に気持ちよく反応すること。動きすぎないように注意すること。

1. **自動ログイン機能:** アクセスされたURLのクエリパラメータ id を検証し、ユーザーを特定してセッションを開始する。
2. **プレイリスト管理:**
   * 任意の名前でフォルダを作成・削除できる機能。
   * フォルダ内にYouTube/ニコニコ動画のURLを追加・削除できる機能。

## **8\. セキュリティと安定性のための対策事項**

1. **オーナー不在時の停止:** サーバー側の API において、イベントオーナーのビートが一定時間確認できない場合は、システムを待機状態にする（デフォルトのBGMを返す等）。
2. **レートリミット:** ビート用APIに対する過剰な連続アクセスを防ぐため、サーバー側で同一IP/同一IDからのアクセス頻度を制限する。
3. **リンク切れ対策:** 動画プレイヤーがエラー（非公開動画など）で停止した場合、Udon側で一定時間後に自動で次の api/get を叩き直すフォールバック処理を実装する。
