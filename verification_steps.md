# LyrisphereWeb 単体検証手順書

本ドキュメントは、VRChatクライアント（Udon）側の実装を待たずに、サーバー側（Web API・マイページ）の機能単体テストをローカル環境で行うための手順をまとめたものです。

---

## 1. サーバーの起動

Dockerを用いて本番相当の環境（Node.jsのバージョン問題を回避した環境）でサーバーを立ち上げます。

1. ターミナル（PowerShellなど）を開き、プロジェクトのルートディレクトリに移動します。
2. 以下のコマンドを実行してコンテナをビルド・起動します。
   ```bash
   docker-compose up -d --build
   ```
3. 起動が完了したら、ブラウザで `http://localhost:3000` にアクセス可能になります。

※ログを確認したい場合は `docker-compose logs -f` を実行してください。

---

## 2. マイページ（UI）の検証

Udon側で生成されるURLを模倣して、マイページに直接アクセスします。
以下のURL（`TestUser` という名前からアルゴリズムで生成した暗号化ID付与済み）をブラウザで開いてください。

**👉 テスト用URL:**
[http://localhost:3000/api/login?id=GBwBHSYDDRc](http://localhost:3000/api/login?id=GBwBHSYDDRc)

### 【確認するポイント】
*   ログイン画面を経由せず、「TestUser さん、おかえりなさい」というマイページが**白基調の洗練されたUI**で開くか。
*   「新しいフォルダを作成」からフォルダが作成できるか。
*   YouTubeのURL（例: `https://www.youtube.com/watch?v=jfKfPfyJRdk`）を追加し、タイトルとサムネイルが自動取得されて表示されるか。

---

## 3. VRChatからのビート送信検証（ターミナル）

VRChatのワールド内から送信される定期的な通信（生存信号）をシミュレートします。
APIは `User-Agent` でブラウザとUdonを区別しているため、`User-Agent` を偽装してリクエストを送ります。

ターミナル（PowerShellなど）で以下のコマンドを実行してください。

```powershell
curl -A "VRChat" "http://localhost:3000/api/login?id=GBwBHSYDDRc"
```

### 【確認するポイント】
*   ターミナルに何も出力されないこと（空の 200 OK が返り、データ転送量を節約する仕様通りに動作しているか）。
*   この通信により、このユーザーが「アクティブ（ワールド内に存在する）」として認識されます。

---

## 4. 楽曲の抽選・リダイレクトAPI (`/api/get`) の検証

動画プレイヤー（Yama Playerなど）が楽曲を読み込む際の挙動を確認します。
**手順3のビート送信を行ってから2分以内**に、以下のURLをブラウザで開いてください。

**👉 抽選API URL:**
[http://localhost:3000/api/get](http://localhost:3000/api/get)

### 【確認するポイント】
*   手順2でマイページに登録したYouTube動画の実際のURLに自動的にリダイレクト（ページ遷移）するか。
*   （※もしビートを一度も送信していない場合や、2分以上経過してアクティブユーザーがいないと判定された場合は、プログラム内に仮置きされているデフォルトBGMにリダイレクトされます）

---

## 5. 楽曲メタデータ取得API (`/api/info`) の検証

動画の再生開始イベント時に、ワールドのUIパネルへ表示する情報を取得するAPIの挙動を確認します。
手順4のリダイレクトを確認した直後に、以下のURLをブラウザで開いてください。

**👉 メタデータAPI URL:**
[http://localhost:3000/api/info](http://localhost:3000/api/info)

### 【確認するポイント】
*   画面にJSON形式で、先ほど抽選で選ばれた楽曲の情報が表示されるか。
*   `title`（曲名）、`provider_name`（提供者: TestUser）、`folder_name`（フォルダ名）が正しく含まれているか。

---

以上の手順により、「自動ログイン → 楽曲登録 → 生存確認 → 動画抽選 → 情報取得」という一連のシステム要件をすべて単独で検証できます。
